# 로그인 기능 기획서

## 개요
사용자가 이메일과 비밀번호를 입력하여 서비스에 로그인하는 기능입니다.

## 기능 요구사항

### 1. 로그인 폼
- 이메일 입력 필드 (필수)
- 비밀번호 입력 필드 (필수)
- "로그인" 버튼
- "비밀번호 찾기" 링크
- "회원가입" 링크

### 2. 이메일 검증
- 이메일 형식이 유효해야 함 (@ 포함, 도메인 포함)
- 최대 255자

### 3. 비밀번호 규칙
- 최소 8자 이상
- 영문, 숫자, 특수문자 중 2가지 이상 포함

### 4. 로그인 처리
- 이메일과 비밀번호가 일치하면 메인 페이지로 이동
- 5회 연속 실패 시 계정 잠금 (30분)
- 로그인 성공 시 JWT 토큰 발급
- "로그인 상태 유지" 체크 시 30일간 토큰 유지

### 5. 에러 처리
- 이메일 미입력: "이메일을 입력해주세요"
- 비밀번호 미입력: "비밀번호를 입력해주세요"
- 이메일 형식 오류: "올바른 이메일 형식을 입력해주세요"
- 로그인 실패: "이메일 또는 비밀번호가 올바르지 않습니다"
- 계정 잠금: "로그인 시도 횟수를 초과했습니다. 30분 후 다시 시도해주세요"
